# Сценарии интеграции

Ниже готовые рабочие схемы под разные типы серверов.

## Сценарий A: Bedrock Dedicated Server (рекомендованный)

Стек:

- `VoiceCraft.Server`
- `VoiceCraft.Addon.Core.McHttp`
- VoiceCraft клиенты у игроков

Шаги:

1. Разверните `VoiceCraft.Server`.
2. В `ServerProperties.json` оставьте `McHttpConfig.Enabled = true`.
3. Откройте доступ BDS к `McHttpConfig.Hostname`.
4. Установите `Core.McHttp` пакеты в BDS.
5. Выполните `voicecraft:vcconnect <hostname> <token>`.
6. Проверьте bind игрока через `voicecraft:vcbind <key>`.

## Сценарий B: Локальный/одиночный мир Bedrock

Стек:

- локальный `VoiceCraft` клиент (с McWss)
- `VoiceCraft.Addon.Core.McWss`

Шаги:

1. Включите McWss в `ServerProperties.json`.
2. Убедитесь, что `DataTunnelCommand = "voicecraft:data_tunnel"`.
3. Установите `Core.McWss` в мир.
4. В мире выполните `voicecraft:vcconnect <token>`.

## Сценарий C: Java + Geyser/Floodgate сеть

Стек:

- `VoiceCraft.Server`
- `GeyserVoice` на Velocity/Bungee
- `GeyserVoice` на backend Paper/Folia

Шаги:

1. На proxy задайте `host/port/server-key` до `VoiceCraft.Server`.
2. На backend Paper включите `server-behind-proxy: true`.
3. Перезагрузите плагин `/voice reload`.
4. Проверьте bind и передачу позиций.

## Пример минимального production-конфига связки (фрагмент)

```json
{
  "VoiceCraftConfig": {
    "Port": 9050,
    "MaxClients": 250,
    "PositioningType": 0
  },
  "McHttpConfig": {
    "Enabled": true,
    "LoginToken": "replace-with-strong-token",
    "Hostname": "http://0.0.0.0:9050/",
    "MaxClients": 10
  },
  "McWssConfig": {
    "Enabled": false,
    "LoginToken": "replace-with-strong-token",
    "Hostname": "ws://0.0.0.0:9051/",
    "DataTunnelCommand": "voicecraft:data_tunnel"
  }
}
```

## Диагностика, если что-то не работает

1. Проверьте токены (`LoginToken`) и соответствие команды `vcconnect`.
2. Проверьте доступность портов и маршрутизацию между узлами.
3. Для proxy-сети проверьте `server-behind-proxy` на backend Paper.
4. Если McWss нестабилен, уменьшите нагрузку и проверьте лимиты команд/длины payload.
5. В крайнем случае временно отключите проблемные packet types в `DisabledPacketTypes` (точечно и осознанно).
