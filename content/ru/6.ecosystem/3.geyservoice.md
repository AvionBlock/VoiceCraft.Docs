# GeyserVoice (Java/Geyser мост)

Репозиторий: [AvionBlock/GeyserVoice](https://github.com/AvionBlock/GeyserVoice)

`GeyserVoice` связывает Java-сервер (Paper/Folia) и прокси (Velocity/Bungee) с `VoiceCraft.Server`.

## Поддерживаемые платформы плагина

По коду:

- Paper/Folia (`io.greitan.avion.paper.GeyserVoice`)
- Velocity (`io.greitan.avion.velocity.GeyserVoice`)
- Bungeecord (`io.greitan.avion.bungeecord.GeyserVoice`)

## Установка

1. Скачайте `.jar` из релиза.
2. Положите jar в `plugins/`:
   - либо на standalone Paper,
   - либо на proxy + backend Paper-серверы.
3. Запустите сервер, чтобы создался `plugins/GeyserVoice/config.yml`.
4. Отредактируйте `config.yml`.
5. Выполните `/voice reload`.

## Конфиг `config.yml`

Блок `config`:

- `debug` — debug-режим плагина.
- `lang` — язык сообщений плагина.
- `host` — хост `VoiceCraft.Server`.
- `port` — порт `VoiceCraft.Server`.
- `server-key` — ключ сервера для логина плагина.
- `auto-reconnect` — авто-реконнект при потере соединения.
- `server-behind-proxy` — важно для Paper за Velocity/Bungee.

Блок `voice`:

- `proximity-distance` — дистанция proximity-логики.
- `proximity-toggle` — включение proximity поведения.
- `voice-effects` — звуковые эффекты.
- `not-in-voice-symbol` / `in-voice-symbol` — символы статуса.
- `send-binded-message` — сообщение при bind.
- `send-disconnect-message` — сообщение при disconnect.
- `send-voip-disconnect-message` — сообщение при закрытии серверного VoIP.
- `send-connection-lost-message` — сообщение при потере подключения.
- `position-task-interval` — тик-интервал отправки позиций.

Блок `players` — служебный кэш autobind-значений.

## Команды `/voice`

По обработчику `BaseVoiceCommand`:

- `connect <host> <port> <key>`
- `reconnect [true|false]`
- `disconnect`
- `settings`
- `bind <key>`
- `bindfake <key> <name>`
- `updatefake <key>`
- `clearautobind`
- `reload`

## Права (permissions)

Из `plugin.yml`/`bungee.yml`:

- `voice.cmd` (default: true)
- `voice.connect` (op)
- `voice.reconnect` (op)
- `voice.disconnect` (op)
- `voice.settings` (op)
- `voice.bind` (true)
- `voice.bindfake` (op)
- `voice.reload` (op)

## Proxy-режим (очень важно)

Если Paper сервер стоит за Velocity/Bungee и плагин установлен на proxy:

1. На Paper ставьте `server-behind-proxy: true`.
2. На Paper не настраивайте `host/port/key` (источник правды — proxy).
3. На proxy держите валидные `host/port/server-key`.

В коде Paper-плагина при `server-behind-proxy: true` reconnect локально не выполняется, и это ожидаемое поведение.

## Технические детали интеграции

- Plugin messaging channel: `geyservoice:main`.
- В proxy-сценарии измерения world/dimension нормализуются с префиксом имени backend-сервера.
- HTTP вызовы к `VoiceCraft.Server` идут с таймаутом 5 секунд.

## Известные ограничения в текущем коде

- `updatefake` сейчас заглушка (реальное обновление fake-пользователя не завершено).
- `settings` команда есть, но логика минимальная/служебная.

## Рекомендуемый production-чеклист

1. Ограничьте `/voice connect|reload` только staff-группам.
2. Включите `server-behind-proxy` только там, где реально есть proxy-слой.
3. Проверьте, что у proxy стабильный маршрут до `VoiceCraft.Server`.
4. Настройте перезапуск прокси-плагина/сервера при сетевых сбоях (systemd/screen/панель).
