# VoiceCraft.Addon (Bedrock Addon)

Репозиторий: [AvionBlock/VoiceCraft.Addon](https://github.com/AvionBlock/VoiceCraft.Addon)

В репозитории 3 рабочих варианта:

1. `Basic` — готовый UX-аддон с bind и меню эффектов.
2. `Core.McHttp` — транспорт через McHttp (под BDS).
3. `Core.McWss` — транспорт через командный data tunnel (singleplayer/world).

## Namespace

Во всех пакетах:

- `VoiceCraft.Namespace = "voicecraft"`

## Команды аддона

### Basic

- `voicecraft:vcbind <binding_key>`
  - permission: `Any`
  - привязывает игрока к сетевой сущности VoiceCraft.
- `voicecraft:vcsettings`
  - permission: `GameDirectors`
  - открывает меню настройки эффектов/игроков.

### Core.McHttp

- `voicecraft:vcconnect <hostname> <token>`
  - permission: `GameDirectors`
  - подключает мир к McHttp endpoint.

### Core.McWss

- `voicecraft:vcconnect <token>`
  - permission: `Host`
  - подключает к McWss.
- `voicecraft:data_tunnel [data]`
  - permission: `Host`
  - двунаправленный туннель пакетов (строки `|`-разделители, порция ~1000 символов).

## Как работает bind-механика (Basic)

По `BindingManager.ts`:

1. Когда появляется новая network entity, ей назначается случайный `binding key` (длина 5).
2. В описание сущности пишется: `Welcome! Your binding key is ...`.
3. Игрок вводит `voicecraft:vcbind <key>`.
4. Сущность привязывается к игроку, имя/описание обновляются.
5. На выходе игрока происходит unbind и генерация нового ключа.

Скрипт-события:

- `voicecraft:onPlayerBind`
- `voicecraft:onPlayerUnbind`

Это удобно для ваших automation-скриптов мира.

## Меню эффектов (Basic)

Через `voicecraft:vcsettings` доступны:

- Visibility
- Proximity
- Directional
- Proximity Echo
- Echo
- Proximity Muffle
- Muffle

Эффекты задаются по `bitmask` через `McApiSetEffectRequestPacket`.

## Что можно кастомизировать

- Логику bind/unbind (например, whitelist по тегам, роли, мини-игры).
- Правила worldId/позиционирования (отправка world id, position, rotation в interval).
- Админские формы (`@minecraft/server-ui`) для контроля активных привязок.

## Ограничения текущей реализации

- В меню действий над игроком (`Kick`) в `Basic` сейчас стоит `Not implemented`.
- Для `Core.McWss` стабильность сильно зависит от лимитов команд и длины payload в серверной конфигурации.

## Рекомендованный порядок настройки для BDS

1. На `VoiceCraft.Server` включите `McHttpConfig.Enabled = true`.
2. Убедитесь, что `McHttpConfig.Hostname` доступен с BDS.
3. Возьмите `LoginToken` из `ServerProperties.json`.
4. Установите пакет `Core.McHttp` в мир/сервер.
5. Выполните `voicecraft:vcconnect <hostname> <token>`.
6. Проверьте bind через `voicecraft:vcbind <key>`.

## Рекомендованный порядок настройки для singleplayer

1. На стороне VoiceCraft включите `McWss`.
2. Установите пакет `Core.McWss`.
3. Выполните `voicecraft:vcconnect <token>`.
4. Убедитесь, что `voicecraft:data_tunnel` не переименован в `ServerProperties.json`.
