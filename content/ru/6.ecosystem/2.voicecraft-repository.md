# VoiceCraft (репозиторий и сборка)

Репозиторий: [AvionBlock/VoiceCraft](https://github.com/AvionBlock/VoiceCraft)

## Что внутри репозитория

- `VoiceCraft.Client/*` — клиенты под разные платформы (Windows/Linux/macOS/Android/iOS/Web).
- `VoiceCraft.Server` — сервер голосового транспорта и API-слой McHttp/McWss.
- `VoiceCraft.Core`, `VoiceCraft.Network` — общие модели, протокол, сетевой слой.

## Требования для сборки из исходников

По коду проекта:

- .NET SDK `9.0.312` (`global.json`, rollForward `latestMinor`).

Проверка:

```bash
dotnet --info
```

## Сборка и запуск

```bash
git clone https://github.com/AvionBlock/VoiceCraft.git
cd VoiceCraft

dotnet restore
dotnet build -c Release
```

Запуск сервера:

```bash
dotnet run --project VoiceCraft.Server -- --language en-US
```

Полезные аргументы root-команды сервера:

- `--language <locale>` — язык логов сервера.
- `--exit-on-invalid-properties` — завершать процесс, если `ServerProperties.json` невалидный.

## Платформенные клиенты (сборка)

Минимально:

```bash
dotnet build VoiceCraft.Client/VoiceCraft.Client.Windows -c Release
dotnet build VoiceCraft.Client/VoiceCraft.Client.Linux -c Release
dotnet build VoiceCraft.Client/VoiceCraft.Client.MacOS -c Release
```

Для мобильных таргетов обычно нужен отдельный toolchain (Android/iOS SDK).

## Практическая настройка под прод

1. Запустите `VoiceCraft.Server` один раз, чтобы сгенерировался `config/ServerProperties.json`.
2. Поменяйте `LoginToken` в `McHttpConfig`/`McWssConfig` на уникальные значения.
3. Явно проверьте `Port`, `Hostname`, `MaxClients`.
4. Откройте только нужные порты на firewall/NAT.
5. Не отключайте packet types в `DisabledPacketTypes`, если не понимаете последствия.

## Связка с аддоном

- Для BDS: обычно `McHttpConfig.Enabled = true`.
- Для singleplayer мира: обычно `McWssConfig.Enabled = true` и используется `voicecraft:data_tunnel`.
